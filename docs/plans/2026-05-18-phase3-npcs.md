# Phase 3 — PNJs (NPCs)

**Objectif :** Les agents Claude définis dans `ledger.json` apparaissent comme des sprites pixel art vivants sur la parcelle. Radion est déjà dans le ledger — il doit marcher sur l'herbe.

**Livrable :** Lancer l'app → le PNJ "Radion" apparaît sur la parcelle centrale, se balade lentement (idle wander), son nom flotte au-dessus de lui. Si `bond ≥ 1`, un indicateur de bond est visible. Le compteur de Bits continue de fonctionner.

---

## Décisions techniques

| Décision | Choix |
|---|---|
| Sprite NPC | `SKSpriteNode` coloré (placeholder), 16×16px centré sur un tile |
| Label nom | `SKLabelNode` flottant au-dessus du sprite, Monaco 9pt |
| Wander | Timer aléatoire, sélectionne un tile voisin libre sur la parcelle, `SKAction.move` |
| State | `idle` (wander) ou `working` (future session active — pas en Phase 3) |
| Source données | `VillageEngine.agents` déjà disponible via `@Observable` |
| NPC layout | 1 NPC par slot, position initiale aléatoire sur la parcelle 20×20 |

## Tâches

### Task 1 — NPCModel

**But :** Struct qui représente l'état d'affichage d'un PNJ, distinct du `AgentRecord` du ledger.

**Fichier :** `Sources/NookApp/NPCModel.swift`

```swift
struct NPCModel {
    let id: String          // = AgentRecord.name (clé ledger)
    var name: String
    var bond: Int
    var totalTokens: Int
    var tileX: Int          // position courante sur la grille (parcelle)
    var tileY: Int
}
```

**Critère de succès :** Compile proprement, importable dans NPCManager.

---

### Task 2 — NPCSprite

**But :** `SKNode` qui affiche un NPC : sprite coloré + label nom + indicateur bond.

**Fichier :** `Sources/NookApp/NPCSprite.swift`

**Spec :**
- Sous-classe de `SKNode`, annotée `@MainActor`
- `init(model: NPCModel)`
- Sprite : `SKSpriteNode`, taille 16×16, couleur `#7EC8A4` (vert-menthe — placeholders pour tous les NPCs Phase 3)
- `filteringMode = .nearest`
- Label nom : `SKLabelNode`, Monaco 9pt, blanc, positionné à y=+14 au-dessus du sprite
- Bond badge : `SKLabelNode` petit (7pt), `⬡ N`, positionné à y=+24, visible seulement si `model.bond >= 1`
- Méthode `update(model: NPCModel)` pour mettre à jour nom/bond sans recréer le node

**Position en world coordinates :** le NPCSprite est positionné par NPCManager selon `tileX`/`tileY`. La conversion tile → world est la même que TileMap :
```swift
x = CGFloat(tileX) * TileMap.tileSize + TileMap.tileSize / 2
y = CGFloat(tileY) * TileMap.tileSize + TileMap.tileSize / 2
```

**Critère de succès :** Instancier `NPCSprite(model:)` et l'ajouter à la scène → sprite coloré visible avec nom flottant.

---

### Task 3 — NPCWander

**But :** Comportement idle : le NPC se déplace vers un tile voisin aléatoire sur la parcelle à intervalles réguliers.

**Fichier :** `Sources/NookApp/NPCWander.swift`

**Spec :**
- Classe `NPCWander`, `@MainActor`
- `init(sprite: NPCSprite, model: inout NPCModel)`
- Méthode `start()` : lance une `SKAction` repeat-forever de wander
- Wander loop :
  1. Attendre un délai aléatoire entre 1.5s et 4.0s (`SKAction.wait(forDuration:withRange:)`)
  2. Choisir un nouveau tile : offset aléatoire de -2 à +2 en X et Y, clampé dans les bounds de la parcelle (origin + width/height de `TileMap.parcelleOriginX/Y/Width/Height`)
  3. Calculer la world position cible
  4. `SKAction.move(to:duration:)` — durée proportionnelle à la distance (~0.4s par tile)
  5. Mettre à jour `model.tileX/tileY`
- Méthode `stop()` : retire les actions du sprite
- Le wander ne traverse pas les bords de la parcelle

**Critère de succès :** Le NPC bouge lentement et aléatoirement sur la parcelle, reste dans les 20×20 tiles d'herbe.

---

### Task 4 — NPCManager

**But :** Synchronise `VillageEngine.agents` → sprites NPC dans la scène.

**Fichier :** `Sources/NookApp/NPCManager.swift`

**Spec :**
- Classe `NPCManager`, `@MainActor`
- `init(scene: SKScene, engine: VillageEngine)`
- Méthode `sync()` : appelée à chaque changement de `engine.agents`
  - Pour chaque agent dans `engine.agents` : créer `NPCModel` si nouveau, mettre à jour si existant
  - Créer `NPCSprite` + `NPCWander` pour chaque nouveau NPC, les ajouter à la scène
  - Position initiale : tile aléatoire dans la parcelle (éviter le centre occupé par la tente)
  - Supprimer les sprites des agents disparus du ledger
- Stocker : `var sprites: [String: NPCSprite]`, `var wanders: [String: NPCWander]`, `var models: [String: NPCModel]`
- Pas de limite de slots en Phase 3 (ça vient en Phase 4+)

**Critère de succès :** Au lancement, Radion apparaît sur la parcelle. Ajouter un agent dans `ledger.json` via atomic write → nouveau NPC apparaît en live.

---

### Task 5 — Intégration dans VillageScene

**But :** Brancher NPCManager à VillageScene et VillageEngine.

**Fichier à modifier :** `Sources/NookApp/VillageScene.swift`

**Changements :**
1. Ajouter `private var npcManager: NPCManager?`
2. Dans `configure(engine:)` :
   - Initialiser `npcManager = NPCManager(scene: self, engine: engine)`
   - Appeler `npcManager?.sync()` immédiatement
3. Dans `update(_ currentTime:)` :
   - Observer les changements de `engine.agents` pour appeler `npcManager?.sync()` si nécessaire
   - Solution simple : stocker `private var lastAgentCount: Int = 0`, si count change → sync
4. Ne pas casser les fonctionnalités existantes (camera, tileMap, HUD SwiftUI)

**Critère de succès :** App relancée → Radion visible sur la parcelle, bouge, nom affiché. Live-reload : modifier ledger.json → NPCs se mettent à jour.
